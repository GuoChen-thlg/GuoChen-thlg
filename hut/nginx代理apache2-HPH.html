nginx反向代理apache2+HPH

279字小于1分钟

nginx 反向代理

2021-03-26T09:45:37.000Z

> 背景：在公司部署项目的时候遇到端口冲突，原本服务器上跑着一个 php 项目使用的是 apache2 。
> 既然我使用nginx进行node反向代理，我也可以反向代理 apache2

解除 443 端口占用

修改 apache2 配置文件 /etc/apache2/ports.conf

将端口修改

...
<IfModule ssl_module>
	Listen 444
</IfModule>

<IfModule mod_gnutls.c>
	Listen 444
</IfModule>
...

修改虚拟机配置文件 在 /etc/apache2/sites-enabled 下

...
<VirtualHost _default_:444>
...

修改完成后 重启 apache2

/etc/init.d/apache2  restart

检查项目是否可以跑通 （加上端口号）

设置 nginx 反向代理

在 /etc/nginx/sites-available 目录下创建一个配置文件 复制默认文件的内容进行修改

...
server {
  ...
	listen 443 ssl default_server;
	ssl_certificate  /etc/nginx/cert/xxx.pem;
	ssl_certificate_key  /etc/nginx/cert/xxx.key;
	server_name  xxx.com;
	location / {
    ...
		proxy_set_header Host $host;
		proxy_set_header X-Real-IP $remote_addr;
		proxy_pass  https://xxx.xxx.xxx.xx:444;
		index index.php index.html index.htm index.nginx-debian.html;
	}
}
...

设置软连接

ln -s /etc/nginx/sites-available/xxx.conf /etc/nginx/sites-enabled/

重启 nginx

nginx -s reload
